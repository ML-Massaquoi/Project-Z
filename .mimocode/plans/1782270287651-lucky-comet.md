# Fix: `employment_type` PostgreSQL Enum Mismatch

## Root Cause

Migration `0002` created the PostgreSQL enum type `employment_type` with values `('permanent', 'contract', 'casual')` and set `server_default='permanent'`. The Python `EmploymentType` enum has completely different values: `('full_time', 'part_time', 'contract', 'intern', 'consultant', 'temporary')`.

Migration `0027` tried to fix this by converting to `VARCHAR(30)`, but it checks `if not column_exists(...)` first — since the column already exists from 0002, the fix is skipped.

**Result**: The database column is a PostgreSQL enum with values `('permanent', 'contract', 'casual')`, but the model uses a Python enum with `('full_time', ...)`. Any query reading a row with `employment_type='permanent'` crashes with `LookupError`.

## Fix Plan

### Step 1: Create migration `0028_fix_employment_type.py`

Convert the column from PostgreSQL enum to plain VARCHAR:

```python
def upgrade():
    # 1. Update legacy data
    op.execute("UPDATE employees SET employment_type = 'full_time' WHERE employment_type = 'permanent'")
    op.execute("UPDATE employees SET employment_type = 'temporary' WHERE employment_type = 'casual'")
    
    # 2. Alter column to VARCHAR(30)
    op.execute("ALTER TABLE employees ALTER COLUMN employment_type TYPE VARCHAR(30) USING employment_type::text")
    
    # 3. Drop old enum type (only if no other columns use it)
    op.execute("DROP TYPE IF EXISTS employment_type")
```

### Step 2: Revert model to `String(30)` (not SAEnum)

In `backend/app/models/employee.py`, change `employment_type` back to plain `String(30)` but keep the `EmploymentType` enum for API validation:

```python
employment_type: Mapped[Optional[str]] = mapped_column(
    String(30), nullable=True, default="full_time"
)
```

### Step 3: Revert `Mapped` type annotation to `Optional[str]`

Since we're using String(30) in the DB, the type annotation should be `Mapped[Optional[str]]` (not `Mapped[Optional[EmploymentType]]`).

### Step 4: Revert `enrollment.py` change

In `backend/app/api/v1/enrollment.py:442`, change back to `emp_type.value if emp_type else None` — this passes a string to the DB, which is correct for VARCHAR.

### Step 5: Revert `employee_service.py` changes

Remove the EmploymentType enum conversion since the column is now String(30).

## Files to modify
